## 功成身退 · 智能回收

### 第〇步：确认值得动手

在开始前快速判断：

```
如果 /ctx-check 已执行过且给出了回收建议 → 直接进入第一步
如果未执行过 /ctx-check → 先提示用户 "建议先运行 /ctx-check 诊断，确认回收收益后再执行"
```

### 第一步：确定当前上下文的关键信息

回顾对话历史，明确以下必须保留的内容：

1. **当前正在进行的任务** — 最近 3 轮对话的完整细节
2. **当前活跃的 Skill** — 最近 3 轮内仍在引用的 Skill 完整内容
3. **关键决策记录** — 用户明确要求记住的事项、技术方案选择
4. **未完成事项** — TODO、待办、用户说"等一下再弄"的内容

### 第二步：确定可以回收的内容

以下内容在压缩时可以放弃完整原文，仅保留一行摘要：

1. **已完成任务的 Skill 内容** — 如用户的审查报告已生成、测试文件已写完
   - 保留摘要格式：`[已完成] 按 {Skill名} 完成 {任务描述}，主要结果：{一句话}`
   
2. **历史中已无关的对话段落** — 前面讨论过、后面再也没提过的话题

3. **已被后续操作覆盖的工具输出** — 如先读了文件，后来直接修改了，原始读取可压缩

### 第三步：执行压缩

在压缩时明确告诉系统：

```
压缩时保留：
- 最近 3 轮对话完整内容
- 当前活跃 Skill 的完整内容
- 关键决策和待办事项

对已完成任务的 Skill：
- [Skill A]：只保留摘要"已按 {标准} 完成 {任务}，主要结果：{一句话}"
- [Skill B]：只保留摘要"..."

功成身退命令本身的提示词也一并回收，只保留摘要"已完成上下文回收"
```

### 第四步：验证与小结

压缩完成后，简要确认：

```
🧹 功成身退 · 回收完成

释放内容：
• [Skill A] — 回收完整内容，保留摘要
• [Skill B] — 回收完整内容，保留摘要

保留内容：
• 最近 3 轮对话
• 当前活跃 Skill：[Skill C]
• 关键决策记录

预估释放：~X tokens
```

### 安全原则

- 🛡️ **有疑问就不回收** — 不确定 Skill 是否真的完成了，就保留
- 🛡️ **当前活跃 Skill 绝不回收** — 哪怕它的内容很长
- 🛡️ **用户明确说"记住"的内容绝不清除**
- 🛡️ **回收的是内容不是记忆** — 摘要行保留了"这件事做过了"的痕迹，用户重新提及时可重新加载
